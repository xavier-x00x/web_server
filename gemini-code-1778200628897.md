# PRD: GopherStack Enterprise (Windows Server Edition)

**Status:** v2.0 - Production Ready  
**Target:** High-Concurrency PHP Environment on Windows Server  
**Author:** Gemini

---

## 1. Ringkasan Eksekutif
GopherStack Enterprise adalah solusi orchestrator infrastruktur web yang dirancang khusus untuk menjalankan stack **Nginx** dan **PHP** di lingkungan Windows Server dengan beban kerja tinggi (1.000+ pengguna concurrent). Proyek ini menggunakan **Golang** sebagai konduktor utama untuk mengatasi keterbatasan arsitektur PHP-CGI pada platform Windows melalui mekanisme *Process Pooling*.

---

## 2. Tujuan Strategis
* **Skalabilitas:** Menangani lebih dari 1.000 koneksi simultan secara stabil melalui load balancing internal.
* **Reliability:** Menyediakan fitur *auto-healing* yang mampu mendeteksi dan memulihkan proses yang gagal secara otomatis.
* **Operational Efficiency:** Memberikan pengalaman deployment satu klik (Zero-Install) dengan konfigurasi otomatis yang dioptimalkan untuk server.

---

## 3. Arsitektur Teknis (High-Concurrency)

### Konsep Utama: Go-as-FPM
Karena Windows tidak memiliki dukungan asli untuk PHP-FPM, Golang bertindak sebagai manager yang:
1.  Melakukan *spawning* sejumlah instans `php-cgi.exe` (Worker Pool).
2.  Mendistribusikan request dari Nginx ke pool tersebut melalui TCP port (e.g., 9000-9100).
3.  Melakukan *recycling* proses jika penggunaan memori melebihi ambang batas.

### Komponen Sistem:
* **Nginx:** Bertindak sebagai Reverse Proxy dan Static File Server.
* **Go Orchestrator:** Mengelola lifecycle proses, monitoring resource, dan automasi Windows Service.
* **PHP Worker Pool:** Kumpulan proses PHP yang siap melayani request dinamis.

## 5. Spesifikasi Non-Fungsional (Production Tuning)

### Performa & Optimasi Windows
* **TCP/IP Tuning:** Go secara otomatis mengoptimalkan nilai `TcpTimedWaitDelay` dan `MaxUserPort` di Windows Registry untuk mencegah *socket exhaustion*.
* **Memory Management:** Membatasi jumlah request per proses PHP (misal: 500 request) sebelum di-recycle untuk mencegah kebocoran memori (memory leak).
* **I/O Optimization:** Menggunakan konfigurasi Nginx yang dioptimalkan untuk Windows (`worker_connections`, `sendfile`, dll).

---

## 6. Keamanan & Isolasi
* **Low-Privilege Execution:** Menjalankan proses worker dengan akun pengguna terbatas (bukan Administrator).
* **Integrated Firewall:** Otomatis mendaftarkan aturan (rules) pada Windows Firewall untuk port 80 dan 443.
* **Request Throttling:** Limitasi jumlah request per IP untuk mencegah serangan brute force sederhana.
